Guidelines & documentation

Consult these resources for contribution standards and deep-dive reading.

  • AGENTS.md Markdown AGENTS.md documents the repository-wide accessibility expectations, Purdue color palette, and tooling boundaries we must honor. It also mandates incremented version comments whenever any file changes, making it the primary governance reference.
  • CLAUDE.md Markdown CLAUDE.md mirrors the ADA and Purdue branding guidance for contributors working through Claude. It reiterates the ban on advanced frameworks and the requirement to annotate every change with updated version headers, reinforcing the shared standards.
  • article.md Markdown article.md contains the “DBA Research Question Handbook,” walking readers from felt business problems to research-ready questions. It expands on frameworks like PICO and SPIDER with practical examples so learners can practice shaping their ideas.
  • researchmethods.md Markdown researchmethods.md offers an in-depth overview of quantitative, qualitative, and mixed-method designs for applied studies. It emphasizes implementation considerations and real-world business applications so DBA students can select fitting approaches.
  • toc.html HTML toc.html provides a Purdue-branded table of contents with succinct file blurbs. It serves as a lightweight directory for quick navigation when you need a scannable alternative to this comprehensive page.

Research question builders

Use these guided wizards to craft and iterate on doctoral-level research questions.

  • RQv1.html HTML RQv1.html is the earliest interactive research question builder, pairing a streamlined wizard with tooltip explanations for variable selection. It focuses on the foundational flow without the automation layers that appear in later versions.
  • RQbuilder6steps.html HTML RQbuilder6steps.html expands the workflow into six stages that embed PICO and SPIDER templates directly on screen. It also adds SMART and FINER suggestion helpers so students can refine their drafts with structured prompts.
  • RQbuilder7steps.html HTML RQbuilder7steps.html further extends the process into seven stages with context capture and variable mapping before synthesis. It auto-prefills framework fields from earlier answers, illustrating how the refined research question evolves.

Method selection tools

Determine appropriate methodologies and supporting assistants for proposed studies.

  • methods.html HTML methods.html offers a wizard-like decision tree that recommends research methodologies after you answer diagnostic questions. It delivers concise guidance once the questionnaire ends, helping researchers converge on a suitable approach quickly.
  • methods-cc.html HTML methods-cc.html is an enhanced edition of the decision tree with expanded methodology coverage and a more sophisticated scoring algorithm. It preserves the same accessible layout while surfacing a broader range of tailored recommendations.
  • AItools.html HTML AItools.html lists Purdue Global AI assistants inside a responsive card grid, highlighting use cases and launch links. It emphasizes skip navigation, focus states, and brand-compliant colors to model an accessible resource catalog.

Case studies

Analyze the CBC TV movie ratings scenario through different accessibility and statistical emphases.

  • case.html HTML case.html hosts the primary TV movie ratings case study complete with dataset generation, hypothesis questions, and statistical calculators. It represents the original experience authored by Dr. Altinoz and balances instruction with guided analysis.
  • caseada.html HTML caseada.html maintains the TV movie ratings scenario while prioritizing accessibility affordances like live regions, semantic controls, and Purdue-compliant styling. It is the go-to version for demonstrating ADA-aligned interactions without altering the core content.
  • casev2.html HTML casev2.html labels the case study as the g-3 release and foregrounds p-value reporting alongside sample-variance t-tests. It keeps the interactive structure while surfacing more statistical context in the status panel for advanced comparisons.

Interactive tools & simulations

Experiment with statistical visualizations and gamified demos.

  • samplesize.html HTML samplesize.html renders a React-powered simulation that visualizes how sample size shapes hypothesis testing outcomes. It combines dynamic charts, conversion lift bands, and accessible summaries to make statistical power concepts tangible.
  • normalcurve.html JavaScript normalcurve.html defines the NormalDistributionExplorer React component so users can adjust the mean, standard deviation, and raw scores. It layers keyboard navigation and live announcements over custom SVG plots of the normal curve.
  • maze.html HTML maze.html delivers “Efe's Monster Maze,” an arcade-style canvas game with scoring, bomb mechanics, and animated overlays. It supports keyboard controls, instruction dialogs, and multi-level progression for playtesting interactive features.
